## What will I learn?

This module focuses on

1. explaining the function of the file systems
2. describe the interfaces to file systems including access methods, file sharing, file blocking and directory structures.
3. introducing the structure of I/O subsystem from hardware to Unix programming
4. describing the performance aspects of I/O Interrupt process for the life cycle of an I/O Request.

This forms the core component to the subject.

## What will I be able to do at the end of this module?

At the end of this module, you should be able to answer these questions:

1. What are the concepts of the layered-structure directory, index, sharing and relative files in file systems?
2. What are the graph directory and acyclic graph? how to avoid the cycles?
3. What are the concepts of the I/O from hardware architecture to its associated Unix programming including the Interrupt process?
4. What is the Direct Memory Access (DMA) and how to run the DMA?
5. What are the Block and Character Devices?
6. What are the Non-blocking and Asynchronous I/O?

## Key vocabulary in this module:

word: layered-structure directory, acyclic graph, I/O hardware architecture, Unix programming to handle the Interrupt process, DMA, Block/Character Devices.

In this module, we have learnt basic concepts of file system interface including the layered-structure directory and acyclic graph in file systems. We have also learnt the I/O concept and the Interrupt handling process. DMA concept and block/character devices.
