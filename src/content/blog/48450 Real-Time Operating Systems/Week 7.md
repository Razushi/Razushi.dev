## What will I learn?

This module focuses on

1. introduction to a detailed description of several examples of organising memory hardware at architecture level.
2. concept description of memory allocation techniques.
3. description of detailed how paging works in contemporary computer systems.
4. key concepts include EAT, memory protection, and shared pages
5. introduction to virtual memory system
6. description of demand paging, page replacement algorithms and allocation of page frames

This forms the core component to the subject.

## What will I be able to do at the end of this module?

At the end of this module, you should be able to answer these questions:

1. What is the local vs physical address space?
2. What is the segmentation architecture, paging and address translation scheme?
3. How to map logical memory to physical memory and what is the concept of shared pages?
4. What is the difference between physical memory and virtual memory?
5. How the page fault occurs and how to handle it?
6. How to calculate the demand paging through the EAT equation?
7. How the page and frame replacement algorithm work (FIFO and LRU)?
 
## Key vocabulary in this module:

word: logical and physical address, STBR/STLR, Page number/offset, EAT, virtual memory, FIFO/LRU for page and frame replacement.

In this module, we have learnt basic concepts of memory and its management strategies including local, vs physical address, physical and virtual memory, paging, address translation, paging fault, EAT/demand paging calculation and page/frame replacement. Now that you're at the end of this module, it's time to review and reflect on what you've learnt.
