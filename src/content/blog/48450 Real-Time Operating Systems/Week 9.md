## What will I learn?

This module focuses on

1. explaining how protection domains, combined with an access matrix, are used to specify the resources a process may access.
2. explaining the implementation of access matrix.
3. discussing security treats and attacks
4. explaining the fundamentals of encryption, authentication, and hashing
5. examining the uses of cryptography in computing.
6. How to conduct the assignments 2 and 3? The tips, framework and programming skills.     
    

## What will I be able to do at the end of this module?

At the end of this module, you should be able to answer these questions:

1. What are the security Violation Categories?
2. How to set up the access matrix with copy rights and owner rights?
3. Can you list the three options for access matrix such as global table, access lists for object and capability list for domains?
4. Can you list the security violation categories?
5. What is the encryption/decryption and how the pair works?
6. What is the symmetric encryption?

## Key vocabulary in this module:

How to access matrix, global table, access lists for object and capability list for domains, encryption/decryption and authentication.

In this module (Protection and Security), we have learnt basic concepts of the protection domains, combined with an access matrix, are used to specify the resources a process may access. We have learnt several access matrix methods including matrix with domains as objects, with copy rights, with owner rights and the control rights. The implementation of access matrix is given to help you go through the examples. In security, we have learnt security violation categories, encryption/decryption, and the authentication.
